# Project Overview

LoreBinders is an AI-powered core engine designed to transform raw manuscripts into structured, searchable, and professional "Story Bibles."

## 🚀 Vision

The goal of LoreBinders is to automate the extraction and analysis of narrative elements (characters, locations, items, etc.) from book manuscripts, ensuring consistency and ease of documentation for authors.

## ✨ Key Features

- **Automated Entity Extraction**: Uses LLMs to identify characters, locations, and other important narrative components.
- **In-depth Character & World Analysis**: Gathers detailed traits, personality, and world-building information.
- **Cohesive Summarization**: Synthesizes gathered data into readable, professional summaries.
- **High-Quality Reporting**: Generates a beautiful Story Bible in PDF format.
- **Interface Agnostic Design**: Can be used via CLI or integrated into web/desktop applications.

## 🛠 Target Audience

- **Authors**: To track series-wide consistency and character development.
- **Editors**: To quickly understand the core world-building elements of a manuscript.
- **Designers**: To have a quick reference guide for story-related visual projects.

## 🛣 Roadmap (Conceptual)

- [ ] **Core Engine Stabilization**: Finalizing the pipeline orchestration and data models.
- [ ] **Advanced Deduplication**: Improving the ability to identify the same entity with varying names/aliases.
- [ ] **Multi-Author Support**: Organizing workspaces by author and series.
- [ ] **Extended Format Support**: Broadening manuscript ingestion capabilities (e.g., DocX).
- [ ] **Web Integration Ready**: Ensuring the API is robust and hooks are ready for frontend integration.
